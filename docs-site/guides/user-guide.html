QuickFW User Guide

Learn to use the QuickFW firewall appliance web interface for monitoring network traffic, viewing security rules, and managing your network.

Introduction

QuickFW is a high-performance L3/L4 stateful firewall appliance designed for small to medium networks. This guide will help you navigate the web interface and perform common user tasks.

What You Can Do

As a user, you can monitor network traffic, view firewall rules and NAT configuration, check system status, and manage your account settings. Administrative tasks like modifying rules require administrator privileges.

System Requirements

  • Modern web browser (Chrome, Firefox, Safari, Edge)
  • JavaScript enabled
  • Network access to the QuickFW appliance
  • Valid user account (default: admin/quickfw)

First Login

When you first access the QuickFW web interface, you'll be presented with a login screen.

Default Credentials

FieldValue
Usernameadmin
Passwordquickfw
Security Notice

For security reasons, you will be required to change the default password on your first login. Choose a strong password with at least 8 characters that is not in the common password list.

Login Process

  1. Open your web browser and navigate to https://<quickfw-ip>
  2. Accept the self-signed TLS certificate (if prompted)
  3. Enter your username and password
  4. If this is your first login, you'll be prompted to change your password
  5. Click "Sign In" to access the dashboard

Dashboard Overview

The Dashboard is your main view into the QuickFW appliance. It provides at-a-glance information about system health, network traffic, and security status.

Dashboard Components

System Status Cards

At the top of the dashboard, you'll see cards displaying:

  • Hostname — System identifier
  • Uptime — How long the system has been running
  • CPU Usage — Current processor utilization
  • Memory Usage — RAM utilization percentage
  • Active Connections — Number of tracked connections

Traffic Graph

The real-time traffic graph shows network activity over time:

  • Blue line — Incoming traffic (RX)
  • Green line — Outgoing traffic (TX)

Recent Events

The events panel shows recent audit log entries, including:

  • Login/logout events
  • Configuration changes
  • System alerts

System Information

The System Information page provides detailed metrics about the QuickFW appliance hardware and software.

Accessing System Info

Active Connections

The Connections page displays the current connection tracking table, showing all active network connections passing through the firewall.

Connection Details

Each connection entry shows:

  • Protocol — TCP, UDP, ICMP, etc.
  • Source — Source IP address and port
  • Destination — Destination IP address and port
  • State — Connection state (ESTABLISHED, TIME_WAIT, etc.)
  • Bytes — Data transferred (if available)

Connection States

StateMeaning
ESTABLISHEDActive, confirmed connection
SYN_SENTConnection initiation sent
SYN_RECVConnection initiation received
TIME_WAITConnection closing, waiting for timeout
CLOSE_WAITRemote closed, waiting for local close
FIN_WAITLocal closed, waiting for remote acknowledgment
Connection Tracking

QuickFW uses Linux conntrack for stateful packet inspection. The connection table has a maximum size limit. When full, new connections may be dropped until existing connections timeout.

Traffic Statistics

Monitor network traffic patterns and identify bandwidth usage across interfaces.

Traffic Metrics

  • Total RX Bytes — Cumulative incoming data
  • Total TX Bytes — Cumulative outgoing data
  • RX/TX Packets — Packet counts
  • RX/TX Errors — Transmission errors
  • RX/TX Dropped — Dropped packets

Real-time Traffic

The dashboard displays real-time traffic graphs showing:

  • Current bandwidth utilization
  • Historical trends (last 30 data points)
  • Peak traffic indicators

Viewing Firewall Rules

The Firewall Rules page displays all configured packet filtering rules, organized by direction (Input, Forward, Output).

Rule Display Format

Each rule shows the following information:

FieldDescription
NameHuman-readable rule identifier
EnabledWhether the rule is active
DirectionInput, Forward, or Output
ProtocolTCP, UDP, ICMP, or Any
SourceSource IP/Network and Port
DestinationDestination IP/Network and Port
ActionAccept, Drop, or Reject
Hit CountNumber of packets matching this rule

Rule Order

Firewall rules are processed in order from top to bottom. The first matching rule is applied, and processing stops. This makes rule order critically important for correct filtering.

Understanding Rule Priority

More specific rules should be placed before general rules. For example, a rule allowing SSH from a specific IP should come before a rule blocking all SSH traffic.

Default Policies

The default policy determines what happens to traffic that doesn't match any rule:

  • Accept — Allow unmatched traffic
  • Drop — Silently discard unmatched traffic

QuickFW defaults to "Drop" for security (default-deny posture).

Changing Your Password

You can change your password from the Account settings page.

Password Requirements

  • Minimum 8 characters
  • Cannot be in the common password list
  • Case-sensitive
Banned Passwords

The following passwords (and their common variants) are not allowed: admin, password, 123456, qwerty, letmein, firewall, changeme, quickfw

Change Password Steps

  1. Navigate to Account → Change Password
  2. Enter your current password
  3. Enter your new password
  4. Confirm your new password
  5. Click "Save Changes"

Session Management

The Account page shows your current session and allows you to log out.

Session Information

  • Username — Current logged-in user
  • Session Expiry — When your session will expire (30 minutes of inactivity)
  • Last Activity — Timestamp of last request

Automatic Timeout

For security, sessions automatically expire after 30 minutes of inactivity. You will be redirected to the login page when this happens.

Manual Logout

To manually end your session:

  1. Click your username in the top-right corner
  2. Select "Logout" from the dropdown menu

Troubleshooting

Cannot Access Web Interface

  • Verify network connectivity to the QuickFW appliance
  • Check that you're using HTTPS (not HTTP)
  • Accept the self-signed certificate if prompted
  • Verify the management interface IP address

Session Keeps Expiring

  • The session timeout is 30 minutes of inactivity
  • Keep the dashboard page open to maintain activity
  • JavaScript must be enabled for session keepalive

Dashboard Shows No Data

  • Wait a few seconds for initial data collection
  • Refresh the page
  • Check browser console for JavaScript errors
  • Verify API connectivity
