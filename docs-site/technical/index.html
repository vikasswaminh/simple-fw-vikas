System Overview

QuickFW is a sophisticated L3/L4 stateful firewall appliance built with Rust, leveraging Linux kernel subsystems (NFQUEUE, nftables, conntrack) for high-performance packet processing.

Verdict Enum (Lines 17-29)

#[derive(Debug)]
pub enum Verdict {
    Accept,           // Accept packet, continue stream
    AcceptModify,     // Accept with modified content
    AcceptStream,     // Accept and stop stream processing
    Drop,             // Drop packet, continue stream
    DropStream,       // Drop packet and block stream
}

Analysis: The Verdict enum provides granular control over packet handling. AcceptStream and DropStream are particularly important for connection tracking efficiency - once a verdict is set for a stream, subsequent packets bypass inspection.

Analysis: The Packet trait abstracts raw packets for processing. Using downcast_rs allows runtime type checking while maintaining trait object safety. The stream_id correlates packets to connections via conntrack.

pub fn generate_firewall_nft_script(config: &FirewallConfig) -> String {
    // Create firewall chains at priority -10 (before DPI queue at 0)
    script.push_str(&format!(
        "add chain {} {} {} {{ type filter hook input priority -10; policy {}; }}\n",
        NFT_FAMILY, NFT_TABLE, FW_INPUT_CHAIN, config.input_policy
    ));
    
    // MGMT_SAFETY bypass for marked packets (lines 398-405)
    for chain in [FW_INPUT_CHAIN, FW_FORWARD_CHAIN, FW_OUTPUT_CHAIN] {
        script.push_str(&format!(
            "add rule {} {} {} meta mark 0x1 accept\n",
            NFT_FAMILY, NFT_TABLE, chain
        ));
    }
    
    // Stateful inspection rules (lines 407-417)
    for chain in [FW_INPUT_CHAIN, FW_FORWARD_CHAIN, FW_OUTPUT_CHAIN] {
        script.push_str(&format!(
            "add rule {} {} {} ct state established,related accept\n",
            NFT_FAMILY, NFT_TABLE, chain
        ));
        script.push_str(&format!(
            "add rule {} {} {} ct state invalid drop\n",
            NFT_FAMILY, NFT_TABLE, chain
        ));
    }
    
    // User rules
    for rule in &config.rules {
        for line in generate_rule_nft(rule, &config.zones) {
            script.push_str(&line);
            script.push('\n');
        }
    }

}

  2. Stateful rules allow established/related, drop invalid

fn create_session(user: &str) -> String {
    // Cryptographically secure random token
    let token: String = OsRng
        .sample_iter(&rand::distributions::Alphanumeric)
        .take(64)  // 64 characters
        .map(char::from)
        .collect();
    
    // Sliding window expiry (30 minutes)
    let now = unix_now();
    sessions.insert(token.clone(), SessionInfo {
        user: user.to_string(),
        last_active: now,
    });
    
    // Evict expired sessions
    sessions.retain(|_, s| now - s.last_active < SESSION_MAX_AGE);
    
    // Cap at 100 sessions
    if sessions.len() >= MAX_SESSIONS {
        let oldest = sessions.iter().min_by_key(|(_, s)| s.last_active);
        // Remove oldest
    }
    
    token
}

Interface Name Validation (Lines 19-31)

static IFACE_RE: LazyLock =
    LazyLock::new(|| Regex::new(r"^[a-zA-Z0-9._\-]{1,15}$").unwrap());

pub fn validate_interface(s: &str) -> Result<(), String> {
    let s = s.trim();
    if s.is_empty() || s == "any" {
        return Ok(());
    }
    if !IFACE_RE.is_match(s) {
        return Err(format!(
            "Invalid interface name '{}': must be 1-15 alphanumeric/._- characters",
            s
        ));
    }
    Ok(())
}

Security: The regex prevents injection by allowing only alphanumeric, dots, underscores, and hyphens. The 15-character limit matches Linux interface name limits.

CIDR Validation (Lines 44-70)

pub fn validate_cidr(s: &str) -> Result<(), String> {
    let s = s.trim();
    if s.is_empty() || s == "any" || s == "*" || s == "0.0.0.0/0" {
        return Ok(());
    }

    if let Some((ip_str, prefix_str)) = s.split_once('/') {
        let ip: IpAddr = ip_str.parse()
            .map_err(|_| format!("Invalid IP address in CIDR '{}'", s))?;
        let prefix: u8 = prefix_str.parse()
            .map_err(|_| format!("Invalid prefix length in CIDR '{}'", s))?;
        let max_prefix = if ip.is_ipv4() { 32 } else { 128 };
        if prefix > max_prefix {
            return Err(format!(
                "Prefix length {} exceeds maximum {} for '{}'",
                prefix, max_prefix, s
            ));
        }
    } else {
        s.parse::()
            .map_err(|_| format!("Invalid IP address '{}'", s))?;
    }
    Ok(())
}

Atomic File Writes (config_utils.rs)

pub fn atomic_write(path: &str, content: &str) -> Result<(), std::io::Error> {
    let temp_path = format!("{}.tmp", path);
    
    // Write to temp file
    std::fs::write(&temp_path, content)?;
    
    // Sync to ensure data reaches disk
    let file = std::fs::File::open(&temp_path)?;
    file.sync_all()?;
    drop(file);
    
    // Atomic rename
    std::fs::rename(&temp_path, path)?;
    
    Ok(())
}

Reliability: Atomic writes prevent corruption from crashes during write operations. The fsync ensures data durability even on power loss.

Key Security Features

  • Self-signed TLS: Generated on first boot with 10-year validity
  • Constant-time password compare: Uses subtle crate to prevent timing attacks
  • Atomic config writes: Prevents corruption during updates
  • Automatic backups: Before any configuration change
  • Re-auth for destructive ops: Password required for reboot/restore
  • CSP headers: Prevents XSS attacks
  • No CORS: API only accessible from same origin
