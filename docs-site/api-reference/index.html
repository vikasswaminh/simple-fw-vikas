API Reference

Complete REST API documentation for QuickFW. All endpoints return JSON and require authentication except where noted.

Base URL

All API requests should be made to: https://<quickfw-host>/api

Authentication

POST /api/auth/login

Description

Authenticate and establish a session. Sets a session cookie on success.

Request Body

{
  "username": "admin",
  "password": "your-password"
}

Response (200 OK)

{
  "token": "session-token-string",
  "expires_in_seconds": 1800
}

// Also sets cookie: quickfw_session=...; HttpOnly; Secure; SameSite=Strict

Errors

401Invalid credentials
429Too many failed attempts (account locked)
POST /api/auth/logout

Description

End the current session and invalidate the session cookie.

Response (200 OK)

{
  "message": "Logged out"
}
POST /api/auth/password

Description

Change the administrator password.

Request Body

{
  "current": "old-password",
  "new": "new-password"
}

Response (200 OK)

{
  "message": "Password changed successfully"
}

Get comprehensive system information including hostname, version, uptime, CPU, and memory usage.

Response (200 OK)

{
  "hostname": "quickfw",
  "version": "1.0.0",
  "uptime_seconds": 86400.5,
  "boot_time": "2026-04-07T10:00:00Z",
  "cpu_usage_percent": 15.2,
  "load_avg_1": 0.25,
  "load_avg_5": 0.30,
  "load_avg_15": 0.28,
  "memory_total_mb": 4096,
  "memory_used_mb": 2048,
  "memory_free_mb": 2048,
  "memory_percent": 50.0
}
GET /api/system/traffic

Description

Get traffic statistics including active connections and interface counters.

Response (200 OK)

{
  "active_connections": 150,
  "total_rx_bytes": 1073741824,
  "total_tx_bytes": 536870912,
  "total_rx_packets": 5000000,
  "total_tx_packets": 3000000
}
POST /api/system/reboot

Description

Reboot the system. Requires password confirmation for security.

Request Body

{
  "confirm_password": "your-admin-password"
}

Response (200 OK)

{
  "message": "Rebooting"
}
GET /api/health

Description

Health check endpoint. Returns OK if the API server is running. No authentication required.

Response (200 OK)

Plain text: OK

POST /api/firewall

Description

Apply firewall configuration. Rules are processed in order. Supports dry-run mode.

Query Parameters

dry_runbooleanIf true, returns nft script without applying

Request Body

{
  "forward_policy": "drop",
  "input_policy": "drop",
  "output_policy": "accept",
  "rules": [
    {
      "name": "Allow HTTP",
      "enabled": true,
      "direction": "forward",
      "protocol": "tcp",
      "dst_port": "80,443",
      "action": "accept"
    }
  ],
  "zones": []
}

Data Types

FirewallRule

FieldTypeRequiredDescription
namestringYesRule identifier
enabledbooleanNoDefault: true
directionstringNoforward, input, output (default: forward)
protocolstringNotcp, udp, icmp, tcp+udp, any
src_ipstringNoSource IP/CIDR or "any"
src_portstringNoSource port(s) or "any"
dst_ipstringNoDest IP/CIDR or "any"
dst_portstringNoDest port(s) or "any"
actionstringYesaccept, drop, reject, log

PortForwardRule

FieldTypeRequiredDescription
protocolstringYestcp or udp
dest_portintegerYesExternal port (1-65535)
forward_tostringYesInternal IP:Port
in_interfacestringYesInbound interface
